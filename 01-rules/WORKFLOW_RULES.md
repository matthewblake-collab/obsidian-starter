# Workflow Rules
*Non-negotiable rules for every AI session. Update when you discover a better way of working.*

## AI session rules

- [e.g. "Targeted edits only — never replace a full file to change a few lines"]
- [e.g. "One task at a time — confirm it worked before the next"]
- [e.g. "Never assume — if unclear, ask ONE clarifying question"]
- [e.g. "Show what you're about to do before doing anything destructive"]

---

## Communication rules

- [e.g. "ELI5 explanations always"]
- [e.g. "Single copiable code blocks — no nested formatting"]
- [e.g. "Answer the question first, then explain"]

---

## Planning rules

- [e.g. "Complex tasks (new screens, data model changes): plan before building"]
- [e.g. "Save plan to file if context gets long, start fresh session to implement"]
- [e.g. "Separate planning from implementation for anything non-trivial"]

---

## Session end rules

- [e.g. "Summarise what was built and rules learned before closing"]
- [e.g. "Update project status after every session that changes something"]
- [e.g. "Copy new learnings to CROSS_SESSION_MEMORY"]

---

## What doesn't work (add when something wastes significant time)

- [e.g. "Planning and implementing in the same session for complex features — always runs out of context"]
