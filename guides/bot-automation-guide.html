Telegram only

What is a Telegram bot? It's an automated account in Telegram you can message like a person. You send it a message, it reads it, and it replies. It can also send YOU messages on a schedule — like a daily briefing. Unlike a real person it's available 24/7 and can be connected to AI.

Step 1 — Open Telegram and search for BotFather:

Open the chat with the verified blue tick.

BotFather will ask you two things: a display name (e.g. "Jarvis") and a username (must end in 'bot', e.g. "my_jarvis_bot"). The username is permanent — pick something you like. The display name can be changed later.

Step 3 — Save your bot token (BotFather gives it to you):

4
Save your tokens as environment variables
Environment variables are like sticky notes your computer keeps permanently. The bot reads them every time it starts. You set them once and never have to type them again.
Terminal

What is an environment variable? Think of it as a saved password your computer remembers. Instead of your bot code containing your secret token (which is dangerous), your computer holds it separately and gives it to the bot when asked. You set it once in a file called .zshrc and it's there forever.

Open your shell settings file in a text editor:
open ~/.zshrc

If it says "file not found", run touch ~/.zshrc first to create it, then run open again.

Add these lines at the bottom — replace with your real values:
# Jarvis / Telegram bot credentials
export TELEGRAM_BOT_TOKEN="paste-your-bot-token-here"
export TELEGRAM_MATT_CHAT_ID="paste-your-chat-id-here"

# Claude Code binary location (run 'which claude' in Terminal to confirm)
export CLAUDE_BIN="$HOME/.local/bin/claude"
Save the file, then reload it:
source ~/.zshrc
Confirm it worked:
echo "Token starts with: ${TELEGRAM_BOT_TOKEN:0:10}"
echo "Chat ID: $TELEGRAM_MATT_CHAT_ID"

You should see the first 10 characters of your token and your chat ID number. If you see blank lines, the source command didn't work — close Terminal, open a new one, and try echo again.

Step 2 — Create the bot file (paste this entire block into Terminal):
cat > ~/jarvis/jarvis_bot.py << 'PYEOF'
#!/usr/bin/env python3
"""Jarvis — AI Telegram bot powered by Claude Code.
Replies to your messages using Claude. Runs daily checks at 8am.
Set TELEGRAM_BOT_TOKEN and TELEGRAM_MATT_CHAT_ID in ~/.zshrc first.
"""
import json, os, subprocess, sys, time, urllib.request, urllib.error
from datetime import datetime, timezone, timedelta

BOT_TOKEN  = os.environ.get("TELEGRAM_BOT_TOKEN", "")
CHAT_ID    = int(os.environ.get("TELEGRAM_MATT_CHAT_ID", "0"))
CLAUDE_BIN = os.environ.get("CLAUDE_BIN", "/Users/" + os.environ.get("USER","") + "/.local/bin/claude")
YOUR_NAME  = os.environ.get("JARVIS_YOUR_NAME", "Boss")
YOUR_WORK  = os.environ.get("JARVIS_YOUR_WORK", "a solo founder building a product")
AEST       = timezone(timedelta(hours=10))  # Change to your timezone offset

# ── Customise this to describe you and your work ──────────────────────────────
SYSTEM_CONTEXT = f"""You are Jarvis, a personal AI assistant for {YOUR_NAME}.
They are {YOUR_WORK}.

Rules:
- Always call them '{YOUR_NAME}'
- Keep replies short and direct — they're messaging from their phone
- If asked to run something in code, explain what it does first
- Personality: calm, capable, no fluff, occasionally dry humour"""

# ── Daily routine: checks for new Anthropic features at 8am ──────────────────
FEATURES_PROMPT = """Search the web and check these sources for any new Anthropic 
features, Claude Code skills, MCP servers, or AI tools announced in the last 7 days:
- https://docs.anthropic.com/en/release-notes/overview
- https://www.anthropic.com/news

For each genuinely new item found, rate its relevance to my specific workflow.
Format your response for Telegram (no markdown headers, plain text):

Anthropic Features Check — [today's date]

[Item name]: [1-line description]
Relevance: HIGH / MEDIUM / LOW — [one sentence why it matters to my work]

If nothing new in the last 7 days, reply with exactly:
No new Anthropic features since last check."""

STATE_FILE = os.path.expanduser("~/jarvis/state.json")

def log(msg):
    print(f"[{datetime.now().isoformat(timespec='seconds')}] {msg}", flush=True)

def tg(method, **params):
    url  = f"https://api.telegram.org/bot{BOT_TOKEN}/{method}"
    data = json.dumps(params).encode()
    req  = urllib.request.Request(url, data=data, headers={"Content-Type": "application/json"})
    with urllib.request.urlopen(req, timeout=30) as r:
        return json.loads(r.read())

def send(text):
    if len(text) > 4000:
        text = text[:3997] + "..."
    try:
        tg("sendMessage", chat_id=CHAT_ID, text=text)
    except Exception as e:
        log(f"send failed: {e}")

def ask_claude(user_text, history):
    convo = ""
    for msg in history[-10:]:
        role = YOUR_NAME if msg["role"] == "user" else "Jarvis"
        convo += f"{role}: {msg['content']}\n"
    prompt = f"{SYSTEM_CONTEXT}\n\nConversation:\n{convo}{YOUR_NAME}: {user_text}\nJarvis:"
    result = subprocess.run(
        [CLAUDE_BIN, "--print", prompt, "--allowedTools", "Read,WebSearch,WebFetch"],
        capture_output=True, text=True, timeout=90, cwd=os.path.expanduser("~")
    )
    if result.returncode != 0:
        raise RuntimeError(f"claude error: {result.stderr[:200]}")
    return result.stdout.strip()

def run_features_check():
    log("Running Anthropic features check...")
    result = subprocess.run(
        [CLAUDE_BIN, "--print", FEATURES_PROMPT, "--allowedTools", "Read,WebSearch,WebFetch"],
        capture_output=True, text=True, timeout=120, cwd=os.path.expanduser("~")
    )
    if result.returncode == 0 and result.stdout.strip():
        send(result.stdout.strip())
        log("Features check sent.")
    else:
        log(f"Features check failed: {result.stderr[:200]}")
        send(f"Features check failed today {YOUR_NAME}. Check jarvis.log for details.")

def load_state():
    try: return json.loads(open(STATE_FILE).read())
    except: return {}

def save_state(state):
    with open(STATE_FILE, "w") as f: json.dump(state, f, indent=2)

def should_run_features_check(state):
    now = datetime.now(tz=AEST)
    if now.hour != 8: return False          # Change 8 to any hour you want
    today = now.strftime("%Y-%m-%d")
    return state.get("last_features_check") != today

def main():
    if not BOT_TOKEN or not CHAT_ID:
        log("FATAL: set TELEGRAM_BOT_TOKEN and TELEGRAM_MATT_CHAT_ID in ~/.zshrc"); sys.exit(1)
    if not os.path.exists(CLAUDE_BIN):
        log(f"FATAL: claude not found at {CLAUDE_BIN}. Run 'which claude' to find it."); sys.exit(1)
    log(f"Jarvis online — watching for messages from chat {CHAT_ID}")
    send(f"Jarvis online, {YOUR_NAME}.")
    offset, history, state = 0, [], load_state()
    while True:
        try:
            if should_run_features_check(state):
                today = datetime.now(tz=AEST).strftime("%Y-%m-%d")
                state["last_features_check"] = today
                save_state(state)
                run_features_check()
            resp = tg("getUpdates", offset=offset, timeout=20, allowed_updates=["message"])
            for update in resp.get("result", []):
                offset = update["update_id"] + 1
                msg    = update.get("message", {})
                if msg.get("chat", {}).get("id") != CHAT_ID: continue
                text   = msg.get("text", "").strip()
                if not text: continue
                log(f"Received: {text[:80]}")
                try:
                    reply = ask_claude(text, history)
                    history.append({"role":"user","content":text})
                    history.append({"role":"assistant","content":reply})
                    history = history[-20:]
                    send(reply)
                except Exception as e:
                    send(f"Sorry {YOUR_NAME}, something went wrong: {e}")
        except urllib.error.URLError as e:
            log(f"Network error: {e} — retrying"); time.sleep(5)
        except Exception as e:
            log(f"Error: {e} — retrying"); time.sleep(5)

if __name__ == "__main__":
    main()
PYEOF

5b
Make Jarvis run automatically in the background forever
Register Jarvis as a Mac background service. It starts at login and restarts automatically if it ever crashes.
TerminalMac only

What is launchd? It's macOS's built-in service manager — the same system that keeps Spotlight, iCloud, and Bluetooth running in the background without you thinking about it. You tell it "run this script at login and keep it alive" and it handles the rest forever.

Create the launchd service file:
cat > ~/Library/LaunchAgents/com.jarvis.bot.plist << 'PLISTEOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.jarvis.bot</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/zsh</string>
    <string>-l</string>
    <string>-c</string>
    <string>python3 /Users/YOUR_USERNAME/jarvis/jarvis_bot.py</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/Users/YOUR_USERNAME/jarvis/jarvis.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/YOUR_USERNAME/jarvis/jarvis-error.log</string>
</dict>
</plist>
PLISTEOF
echo "Created plist file"

Replace YOUR_USERNAME with your actual Mac username. Run echo $USER in Terminal to find it. You need to update it in 4 places in the file above.

Register and start it:
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.jarvis.bot.plist
launchctl list | grep com.jarvis.bot

You should see a line with com.jarvis.bot in the output. A number in the first column means it's running. A dash means it ran and exited — check jarvis.log for errors.

Check it's working and see the live log:
tail -f ~/jarvis/jarvis.log
If you need to restart Jarvis (e.g. after editing the bot file):
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.jarvis.bot.plist
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.jarvis.bot.plist

Part 3 — Daily Routines & The Anthropic Features Check · Steps 9–11
9
How Jarvis's daily routines work
Jarvis runs a loop every 30 seconds checking for messages from you. Once per day at 8am it also runs a special routine automatically — no trigger required from you.
Built in

How does it know what time it is? Every 30 seconds the bot wakes up, checks if it's 8am in your timezone, and checks whether it already ran today. If both are true, it runs the routine. If either is false, it goes back to sleep. It saves "last ran today" to a file so it doesn't run twice. Simple but bulletproof.

The routine logic in plain English:
Every 30 seconds, Jarvis asks:

1. Is it 8am in my timezone right now?
   → If no: skip, check messages, sleep 30s, repeat
   → If yes: go to question 2

2. Did I already run today's check?
   → Check ~/jarvis/state.json for today's date
   → If already ran: skip
   → If not yet ran: run the features check now

3. Run features check:
   → Call Claude with a web search prompt
   → Claude searches Anthropic's release notes
   → Claude summarises what's new and rates relevance to your work
   → Jarvis DMs you the result on Telegram

4. Save today's date to state.json so it won't run again today
To change what time the routine runs — edit jarvis_bot.py:
# Find this line in jarvis_bot.py:
if now.hour != 8: return False

# Change 8 to any hour (0-23):
if now.hour != 7: return False   # runs at 7am
if now.hour != 20: return False  # runs at 8pm
To change your timezone — find AEST and change the offset:
# Find this line in jarvis_bot.py:
AEST = timezone(timedelta(hours=10))

# Change the number to your UTC offset:
timezone(timedelta(hours=0))    # London / GMT
timezone(timedelta(hours=-5))   # New York / EST
timezone(timedelta(hours=-8))   # Los Angeles / PST
timezone(timedelta(hours=1))    # Paris / CET
timezone(timedelta(hours=10))   # Sydney / AEST

After editing jarvis_bot.py, restart Jarvis: launchctl bootout then launchctl bootstrap (same commands as Step 5b).

Add your own daily routines — any time, any task

Customise freely

A routine is just a prompt that runs at a specific time. You write a prompt that tells Claude what to check or create, add it to jarvis_bot.py with an hour, and Jarvis runs it forever. You can have one routine or twenty. Each one runs once per day at its designated hour and DMs you the result.

Template for adding a new routine — paste into jarvis_bot.py:
# Add your prompt (describe what you want Claude to check or write):
MY_ROUTINE_PROMPT = """[Your prompt here. Examples:
- Check these 3 competitor websites and summarise any new features
- Write me 5 social media post ideas about [your topic] for today
- What is today's top news in [your industry]?
- Summarise my last 3 Obsidian session notes and identify patterns]"""

# Add a check function (copy the features check pattern):
def run_my_routine():
    log("Running my custom routine...")
    result = subprocess.run(
        [CLAUDE_BIN, "--print", MY_ROUTINE_PROMPT,
         "--allowedTools", "Read,WebSearch,WebFetch"],
        capture_output=True, text=True, timeout=120,
        cwd=os.path.expanduser("~")
    )
    if result.returncode == 0:
        send(result.stdout.strip())

# Add this inside the while loop (after the features check block):
now = datetime.now(tz=AEST)
if now.hour == 9 and state.get("last_my_routine") != now.strftime("%Y-%m-%d"):
    state["last_my_routine"] = now.strftime("%Y-%m-%d")
    save_state(state)
    run_my_routine()

When something breaks — the 5 most common fixes

Almost every problem falls into one of these categories. Check from top to bottom.

Problem 1
Bot doesn't reply — check: tail -f ~/jarvis/jarvis.log to see if it's running
Problem 2
Claude not found — run: which claude in Terminal and update CLAUDE_BIN in ~/.zshrc
Problem 3
Token errors — re-run: echo $TELEGRAM_BOT_TOKEN to confirm it loaded from .zshrc
Problem 4
Routine not firing — check the timezone offset matches your local time
Problem 5
Slack bot can't post — run /invite @botname in the Slack channel first

What is Claude Code? It's a command-line AI tool made by Anthropic that runs on your computer. Unlike claude.ai in a browser, it can directly read and write files on your Mac. If you have it, this prompt will build your entire Jarvis setup — the bot file, the plist, the config — and start everything for you automatically.

RUN IN Terminal first:
cd ~ && claude

Set up a complete Jarvis bot setup for me. My details:
- My name (what Jarvis calls me): [your name or "Boss"]

- My timezone UTC offset: [e.g. +10 for Sydney, -5 for New York, 0 for London]
- Telegram bot token: [paste your token from BotFather]
- Telegram chat ID: [paste your chat ID from @userinfobot]

Tasks in order:
1. Create ~/jarvis/ directory
